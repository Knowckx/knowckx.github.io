
### 结论
静态网站的最佳去处就是 `EdgeOne Pages`
他能提供给你100~200ms的访问速度 基本和国内差不多了

- 千万别把静态网站发布在`githubPage`上 因为国内访问不了的


### 完整网络流程
Cloudflare DNS 灰色小云朵 + EdgeOne Pages

- 用户在浏览器输入 blog.knowckx.top 
- 因为ns是在cf这里，所以一定会在Cloudflare中进行第一次DNS解析(这是固定成本无法避开)
- 因为配置的是灰色小云朵 Cloudflare只是简单地告诉用户 你去找 EdgeOne 这个地址吧  xx.blog.knowckx.top.dnsoe3.com.
- 用户获得 EdgeOne IP(通常是东京/香港等地的IP)

### 优势: 
避开了 Cloudflare 的“绕路”问题： 流量不经过 Cloudflare 遍布全球但国内访问较慢的节点，只利用了 Cloudflare 毫秒级的 DNS 解析能力。
利用了腾讯云的优质线路： 腾讯云（EdgeOne）的香港/境外节点针对中国大陆有专门的线路优化（如 CN2 或直连），相比 GitHub Pages 的服务器，延迟更低、丢包率极小。
双重保障： Cloudflare 提供了非常稳健的 DNS 服务（不会宕机），EdgeOne 提供了针对国内优化的 CDN 加速。
目前的流程可以简化为：
用户 -> Cloudflare (只问路) -> EdgeOne 边缘节点 (直接取货) -> 展示页面
