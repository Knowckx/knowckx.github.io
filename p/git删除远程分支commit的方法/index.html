Git删除远程分支commit的方法

2022-06-01
阅读时长: 2 分钟

太长不看版

  1. 找到你需要去掉的commit的前一次commit
  2. 执行 git rebase -i commit_id -X their
  3. vi里drop掉不需要的commit
  4. git push -f 覆盖远程分支

原文

最近遇到一个问题，我们的项目代码里把一些密码配置明文传到了github，
但是公司的安全策略不希望我们在代码里直接留有敏感的账号凭证信息，否则会被扫出来合规性问题。
按照我同事之前的经历：他们扫出来之后就会发邮件给你老板，让你连夜起来马上消掉。

因此产生了一个需求，怎么把github上的敏感信息抹掉呢？ 我调研了下，目前已知有几种方式：

  • 通过git revert commit_id来实现
    git revert 可以撤销某次操作，他是通过提交一次新的commit来回滚之前的一次commit的内容，
    大部分情况下这个功能是够用的。
    我们的场景比较特别，因为旧的commit并不会消失，打开旧的commit还是可以看到密码信息，不合规。

  • 通过git reset --hard commit_id直接重置到密码提交前的那一次commit，然后git push -f重置分支
    这样做的问题是需要提交密码的那一次commit是最近发生的，
    不然就需要手动把后面的历史commit都补回来，工作量太大

  • 通过git rebase -i来实现丢弃一个commit
    这是今天发现的解决方案，有点类似于以前git合并commit的操作，比较有趣。

下面说下流程：

找到你需要丢弃的commit_id

在我的例子里，我在bdfb32a这次提交里放了密码

这次commit的前一次是0119c21

所以执行 git rebase -i 0119c21 -X their

  • -i 进入交互模式
  • -X their 方便后续的commit自动合并，不然你需要手动操作冲突。

手动drop

这时进入了vi的编辑界面，此时git会把commit_id之后发生的commit列出来，
我们按i进入编辑，手动把不需要的那个commitbdfb32a前面的pick改成drop.

然后vi和rebase的基本操作：

  1. esc, :wd
  2. git add .
  3. git rebase --continue

手动push 覆盖远程分支

此时你通过git log -10应该可以看到commit的历史已经改变了，bdfb32a已经消失了。

执行git push -f 去github看一下

0119c21之后的这次bdfb32a已经抹除。

收工~！
